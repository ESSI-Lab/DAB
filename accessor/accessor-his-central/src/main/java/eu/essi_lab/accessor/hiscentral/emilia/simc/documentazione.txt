Introduzione
Le API per la consultazione dei dati osservati delle stazioni meteo di Arpae-SIMC sono
basate su Python Eve, la cui documentazione esaustiva è disponibile al seguente URL:
https://docs.python-eve.org/, dove è possibile, in particolare, trovare informazioni in merito a:
-​ Struttura e formato delle risposte
-​ Filtrare le risorse
-​ Selezionare solo alcuni campi di ogni elemento della risorsa (projection)
-​ Ordinare i risultati
-​ Paginazione e numero massimo di risultati per pagina
Paginazione
La paginazione merita un paragrafo a parte. Tutte le risorse usano la paginazione ma, per
questioni di prestazioni, le risposte alle query non contengono il numero totale dei record né
i link all’ultima pagina della query. Si deve quindi iterare sulla pagine, incrementando il
parametro GET page fino a che la risposta non è vuota:
Python
# Esempio di uso della paginazione: itero da pagina 1 fino a che non ottengo

Formato della data
Le date nelle risorse sono espresse nel formato RFC1123 (ex RFC 822), sia nei filtri delle
richieste che nelle risposte. Ad esempio: Tue, 02 Apr 2013 10:29:13 GMT.
Interpretazione dei reftime
Il reftime indicati nella risorse si riferiscono all’istante di validità, quindi all’istante finale
dell’osservazione. Ad esempio, nel caso di una precipitazione oraria dalle 13:00 alle 14:00
del giorno X, il reftime corrisponde alle 14:00 del giorno X. Nel caso di precipitazione
giornaliera dalle 00 del giorno X alle 00 del giorno X+1, il reftime corrisponde alle 00 del
giorno X+1.

-​ reftime_start: istante del primo dato

-​lower_level: livello minimo per cui applicare l’equazione

Dati osservati
Ogni risorsa che contiene dei dati osservati (il cui elenco è disponibile nella risorsa descritta
in questo paragrafo) è composta da un documento per ogni osservazione. Ogni documento
ha i seguenti campi:
-​ _id: identificativo (composto dal concatenamento dell’id della stazione e del reftime
dell’osservazione)
-​ station_id: id della stazione
-​reftime: istante di validità dell’osservazione, cioè l’istante finale.
-​value: valore dell’osservazione o null se non valida
-​_updated, _created: campi builtin di Python Eve con la data di ultimo
aggiornamento e creazione del documento
Esempio di utilizzo delle risorse
Di seguito, un esempio in Python: lo script seleziona le stazioni con quota superiore a 600
metri e stampa a video i dati di precipitazione oraria (dataset
meteo_opendata_stazioni_precipitazione_oraria) con istante di validità tra le
14:00 UTC e le 16:00 UTC del 3 gennaio 2026.Python
import requests
from datetime import datetime, timezone
import itertools
import json
BASE_URL = "https://apps.arpae.it/REST"
DATASET = "meteo_opendata_stazioni_precipitazione_oraria"
DATETIME_FORMAT = "%a, %d %b %Y %H:%M:%S GMT"
REFTIME_START = datetime(2026, 1, 3, 14,
tzinfo=timezone.utc).strftime(DATETIME_FORMAT)
REFTIME_END = datetime(2026, 1, 3, 18,
tzinfo=timezone.utc).strftime(DATETIME_FORMAT)
stations = []
for page in itertools.count(1):
resp = requests.get(f"{BASE_URL}/meteo_opendata_stazioni", params={
"where": json.dumps({
"height": {
"$gt": 600
}
}),
"sort": "_id",
"max_results": 200,
"page": page,
})
resp.raise_for_status()
items = resp.json()["_items"]
if len(items) == 0:
break
else:
stations += items
for station in stations:

resp = requests.get(f"{BASE_URL}/{DATASET}", params={
"where": json.dumps({
"reftime": {
"$gt": REFTIME_START,
"$lt": REFTIME_END,
},
"station_id": station["_id"],
}),
"sort": "reftime",
"max_results": 200,
"page": page,
})
resp.raise_for_status()items = resp.json()["_items"]
